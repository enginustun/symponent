symponent Documentation Version 1.0

Introduction

symponent is a light-weight but very powerful JavaScript UI library that allows you to create reactive UIs with a very small learning curve.
It aims to render only related elements if it is necessary.

Compatibilities

symponent supports IE9 and above, because it uses ECMAScript 5 features.

Prerequisites

This guide assumes intermediate level knowledge of HTML, CSS, and JavaScript. If you think you are not familiar with them, it might not be a good idea to start learning this library. After experiencing the basics you might be symponent master. Prior experience with other frameworks helps, but is not required.

How symponent Works

symponent has one-way data binding. It means you need to update your model data on user input such as if select element's selection changes, if user clicks checkbox, types input to text box, etc. by using events.

symponent allows you to use models within DOM elements with curly brackets while creating these elements(e.g. {modelName.propertyName}). As you can understand that you should pass model parameter to the element created for using it in element.

When you use template syntax and use model in it, symponent automatically binds models and DOM elements. This bindings are only attrbute/value level. It means that when you change models' any property binded to a DOM element, these elements will be updated. "Will be updated" doesn't mean that element will be fully re-rendered. Where you have used these properties in element(e.g. id attribute, value attribute or innerText) only related part will be re-assigned based on models' properties' values.

Interactivity

Interactive user inputs can be defined easily. If you are not familiar with how to use events, we suggest you to check it out Using Events section firstly.

model: { user: userObject } | This syntax creates model scope and allows you to use this model in every child of defined element and in itself.
element.model.user.name = event.target.value; | element.model.user refers to user object of defined model above.
If you change model's user object's property or original userObject's property which is binded to a DOM element, this element will be updated immediately.

Note

Please see API documentation for usage of events.

events: {
    input: function(event, element){
        element.model.user.name = event.target.value;
    }
}
